개요

I&VIEW COMMUNICATION Publishing Design System, 이하 PDS는 웹사이트를 개발할 때 사용자 인터페이스(UI)를 효율적으로 구축하고,
일관성 있게 유지하기 위한 통합적인 가이드와 도구 모음이다. 퍼블리싱 UI 디자인 시스템을 구성하는 주요 요소들과 그 목적은 간략히 아래 내용으로 요약할수 있다.

  • 1. 디자인 원칙 정의
    • 디자인 시스템의 기반을 이루는 핵심 가치와 원칙을 정립하기 위함이다.
  • 2. 컴포넌트
    • 재사용 가능한 UI 요소들을 미리 정의하고 표준화하며, 버튼, 입력창, 라벨 등의 디자인과 스타일을 미리 지정하여 개발력을 향상시키고 일관성을 유지하기 위함이다.
  • 3. 스타일 가이드
    • 색상 팔레트, 타이포, 아이콘과 그리드 시스템 등 전체 디자인에서 사용되는 기본 요소들은 일관되게 사용될 수 있도록 가이드하기 위함이다.
  • 4. 문서화
    • 디자인 원칙, 컴포넌트, 스타일 가이드 등을 활용하는 방법을 자세하게 기술하여, 팀원들이 쉽게 참고할 수 있는 형태로 문서화 하며, 이를 통해 퍼블리싱 작업의 효율성을 높이기 위함이다.
  • 5. 툴킷 및 라이브러리
    • 개발자들이 효과적으로 디자인 시스템을 적용할 수 있도록 다양한 디자인 및 개발 도구를 제공하며, 디자인 툴킷과 프레임워크, 코드 라이브러리 등을 통해 퍼블리싱 작업을 효율적으로 진행하기 위함이다.

이러한 요소들을 종합하여 퍼블리싱 UI 디자인 시스템을 구축하면, 팀원 간의 협업이 원활해지고 디자인과 개발 과정에서 발생할 수 있는 오류를 최소화 하고,
일관된 사용자 경험을 제공함으로써 궁극적으로 브랜드의 인지도와 만족도를 높임에 목표를 둔다.

퍼블리싱 작업 규칙

File Creation Guide

  • 1. html 파일 관리
    • 메인과 관련된 html 마크업 파일은 폴더명 /inv_main에 위치한다
    • 서브와 관련된 html 마크업 파일은 폴더명 /inv_sub에 위치한다
  • 2. Image
    • 폴더명 /img에 모든 이미지를 저장 민 관리한다.
    • favicon 및 logo성 이미지는, /img 폴더내부 폴더 /logo에 위치한다.
    • 사용 되는 모든 비트맵 이미지 포맷은, webp로 저장 및 활용한다.
    • 메인콘텐츠로 사용될 이미지의 파일명은, 앞에 main_cont_img_를 기입하여 사용한다.
    • 메인콘텐츠로 사용될 Background이미지의 파일명은, 앞에 main_bg_img_를 기입하여 사용한다.
    • 서브콘텐츠로 사용될 이미지의 파일명은, 앞에 sub_cont_img를 기입하여 사용한다.
    • 서브콘텐츠로 사용될 Background이미지의 파일명은, 앞에 sub_bg_img_를 기입하여 사용한다.
    • 모든 이미지 파일명 끝에 Numbering을 해준다.
  • 3. Font
    • 폰트는 Linking Style Sheet 형식으로, 한 css에서 관리 및 사용하며, 해당 링크는 아래와 같다.
      - https://inviewdev.github.io/I-VIEW_PDS/css/font.css
